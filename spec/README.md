# spec/

This directory contains formal and semi-formal specifications for Reikan subsystems.

## Contents

Specifications are organized by subsystem:

- `spec/capability/` — Formal capability model and proof obligations
- `spec/memory/` — Memory object model and region invariants
- `spec/language/` — Systems language formal grammar and type rules
- `spec/ipc/` — IPC protocol definitions and message schemas
- `spec/scheduler/` — Scheduling invariants and fairness properties
- `spec/security/` — Threat model and security invariant definitions

## Status

No specifications have been written yet. This directory will be populated as Phase 0 (architecture definition) progresses.

See [docs/architecture-outline.md](../docs/architecture-outline.md) for the full design scope.
